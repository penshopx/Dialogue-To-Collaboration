Sesuai saranku, langkah terbaik berikutnya adalah mengubah blueprint ini jadi prompt Replit "Fase Kolaborasi" — karena Library-mu sudah penuh, yang perlu dibangun tinggal lapisan orkestrasi (Registry + Workroom + Gate + Log). Ini melanjutkan engine generik dari Fase 0 yang sudah kita rancang, jadi nyambung langsung dengan aplikasi MVP-mu.
Berikut prompt siap tempel ke Replit:
 PROMPT REPLIT — FASE KOLABORASI (Lapisan Orkestrasi)
Konteks untuk Replit: Lanjutkan aplikasi yang sudah ada (engine generik App_Types, Workroom_Types, Template, Workrooms, Tasks, Documents, Auth/Branches/Users/Companies). Sekarang tambahkan lapisan orkestrasi kolaborasi multi-agen dengan gerbang manusia (human gate) dan pencatatan keputusan. Jangan hapus tabel lama; tambahkan tabel baru di bawah ini dan saling hubungkan.
A. Tabel baru
1. Agents_Registry (Role Library — kartu peran tiap Claw)
id, nama_agen, slug_url (mis. /qs-claw), kategori (Tender/Teknik/K3-Mutu/Sertifikasi/Manajerial/Energi/Pendidikan/Bisnis), tipe (enum: dialog  / spesialis  / multi_agen ), domain, input_dibutuhkan (text), output_dihasilkan (text), fungsi_peran (enum: strategis/skeptis/eksekutor/narasumber/pack_compiler), aktif (bool)
2. Workroom_Templates (5 template siap jual)
id, nama_template, sektor, deskripsi, output_akhir, mode (enum: profesional/organisasi), aktif
3. Template_Roles (peran di tiap template → agen pengisi)
id, template_id (FK Workroom_Templates), nama_peran, fungsi_peran (strategis/skeptis/eksekutor/dst), agent_id (FK Agents_Registry), urutan
4. Workflow_Stages (pipeline baku, bisa di-reuse)
id, template_id (FK), urutan, nama_tahap (Intake/Framing/Skeptic Gate/Blueprint/Delivery/QA Gate/Release/Retro), tipe_tahap (enum: kerja/gate), pola_operan (enum: ai_first_review/human_first_assist/paralel/agen_ke_agen_gate)
5. Collaborations (instance workroom yang dibuat user)
id, judul, template_id (FK), branch_id (FK Branches), company_id (FK Companies, nullable), pemilik_user_id (FK Users), status (enum: intake/berjalan/menunggu_gate/selesai/dibatalkan), tahap_aktif_id (FK Workflow_Stages), mode, created_at
6. Collaboration_Roles (anggota tim workroom aktif)
id, collaboration_id (FK), nama_peran, fungsi_peran, agent_id (FK, nullable), human_user_id (FK Users, nullable), is_pic (bool)
7. Tasks (operan kerja — perluas tabel lama jika ada)
id, collaboration_id (FK), stage_id (FK Workflow_Stages), judul, penanggung_jawab_role_id (FK Collaboration_Roles), status (enum: to_do/berjalan/menunggu_review/selesai/diblokir), input_brief (text), output_hasil (text), urutan
8. Gates (gerbang manusia ◆ — inti governance)
id, collaboration_id (FK), stage_id (FK), nama_gate (Skeptic Gate/QA Gate), status (enum: terbuka/disetujui/ditolak/revisi), keputusan_oleh_user_id (FK Users, nullable), catatan_keputusan (text), waktu_keputusan (datetime, nullable)
9. Decision_Log (jejak keputusan — wajib, tak bisa dihapus)
id, collaboration_id (FK), gate_id (FK, nullable), aktor (text: nama user/agen), tipe_aksi (enum: usulan/tantangan_skeptis/keputusan_gate/revisi/rilis), ringkasan (text), timestamp (auto)
10. Deliverables (Pack Compiler — artefak final)
id, collaboration_id (FK), nama_artefak, tipe (dokumen/risk_register/laporan/checklist), isi_atau_link (text), versi, disetujui (bool), created_at
B. 6 Form (input no-code → workroom jadi)
Buat wizard 6 langkah yang menulis ke tabel di atas:
Collaboration Spec → buat baris Collaborations (judul, output, batasan, mode).
Team Builder → pilih Workroom_Templates ATAU rakit manual dari Agents_Registry → isi Collaboration_Roles. Wajib ada 3 peran generik (strategis/skeptis/eksekutor).
Workflow Builder → pilih pola_operan per tahap → isi Workflow_Stages.
Human Gates → tentukan tahap mana jadi tipe_tahap=gate + siapa PIC penyetuju.
Logging & Evidence → pilih aksi apa yang dicatat ke Decision_Log.
Deliverables → tentukan bentuk Deliverables akhir + siapa pack compiler.
C. Engine inti
createCollaborationFromTemplate(template_id, judul, branch_id) — clone Template_Roles → Collaboration_Roles, clone Workflow_Stages, set status=intake, tulis baris awal Decision_Log.
Mesin tahap: saat semua Tasks di tahap kerja selesai → pindah ke tahap berikut. Jika tahap berikut tipe_tahap=gate → set Collaborations.status=menunggu_gate, buat baris Gates status terbuka, kunci progres sampai PIC menekan Setujui/Tolak.
Aturan gate (◆): alur tidak lanjut tanpa keputusan manusia. Setiap keputusan otomatis menulis Decision_Log.
Pack Compiler: di tahap Release, kumpulkan semua output_hasil Tasks + keputusan jadi Deliverables.

Dashboard — jumlah workroom per status, gate yang menunggu keputusan, waktu rata-rata ke-rilis.
