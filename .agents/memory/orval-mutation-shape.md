---
name: Orval mutation body shape
description: Orval-generated React Query mutation hooks wrap the request body in { data: InputType }, not the raw input type.
---

All mutations generated by Orval with `customFetch` expect variables shaped as `{ data: BodyType<InputType> }` (and path params separately), NOT the raw input directly.

**Rule:** When calling `.mutateAsync(...)` on any Orval-generated hook, always wrap the request body:
```ts
// CORRECT
createWorkroom.mutateAsync({ data: { name, templateId } })

// WRONG — causes TS2353
createWorkroom.mutateAsync({ name, templateId })
```

For mutations with path params + body:
```ts
updateWorkroomStage.mutateAsync({ workroomId, stageId, data: { ... } })
createWorkroomTask.mutateAsync({ workroomId, data: { ... } })
updateTask.mutateAsync({ id, data: { ... } })
```

**Why:** This is the Orval default behavior for the `customFetch` mutator pattern. The generated type is `{ data: BodyType<InputType> }` where `BodyType` = `InputType` by default.

**How to apply:** Any time you add a new mutation call using a generated hook, wrap the body in `{ data: ... }`. Check the generated hook's TypeScript signature in `lib/api-client-react/src/generated/api.ts` when unsure.
